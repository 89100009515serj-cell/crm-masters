# Первый запуск CRM Мастеров — пошагово для новичка

Это инструкция «с нуля». Иди строго по порядку, не пропуская шаги. Если на каком-то шаге
видишь ошибку — не паникуй, в конце есть раздел «Если что-то не работает» с разбором типичных
проблем.

---

## Шаг 0. Что вообще нужно установить на компьютер

Тебе понадобятся три программы. Поставь их все, прежде чем продолжать.

### 1. Node.js
Это среда, в которой будет работать приложение.

- Зайди на https://nodejs.org
- Скачай версию **LTS** (там будет крупная зелёная кнопка с надписью LTS)
- Установи как обычную программу (next → next → install)
- Проверка: открой терминал (см. ниже, что это) и введи:
  ```
  node -v
  ```
  Должна появиться версия вроде `v20.x.x` или `v22.x.x`. Если появилась — всё ок.

### 2. Редактор кода (VS Code)
Не обязателен для запуска, но тебе будет проще ориентироваться в файлах.
- https://code.visualstudio.com — скачай и установи.

### 3. Терминал — что это и как открыть
Терминал — это окно, куда вводят текстовые команды вместо кликов мышкой.

- **Windows**: нажми Win, напиши "PowerShell", открой его.
  Либо открой VS Code → меню "Terminal" → "New Terminal" (так даже удобнее).
- **Mac**: Cmd+Space, напиши "Terminal", открой.
  Либо тоже через VS Code → Terminal → New Terminal.

Дальше все команды я буду давать как текст в сером блоке — это то, что нужно набрать
в терминале и нажать Enter.

## Шаг 1. Распаковать проект

1. У тебя есть файл `crm-app.zip`, который я прислал. Скачай его на компьютер.
2. Распакуй архив в удобное место, например на Рабочий стол. Получится папка `crm-app`.
3. Открой эту папку в VS Code: меню File → Open Folder → выбери `crm-app`.
4. Открой терминал прямо внутри VS Code (Terminal → New Terminal) — он автоматически
   откроется уже "внутри" этой папки, это важно: все команды ниже нужно вводить именно
   находясь внутри папки `crm-app`.

Проверь, что ты в нужном месте:
```
ls
```
(на Windows в PowerShell тоже работает `ls`, или используй `dir`)

Ты должен увидеть список файлов вроде `package.json`, `prisma`, `src`, `README.md`.
Если видишь это — ты в правильной папке.

## Шаг 2. Установить зависимости проекта

Это скачает все библиотеки, от которых зависит приложение (Next.js, Prisma и т.д.).

```
npm install
```

Это займёт 1-3 минуты, появится много текста — это нормально. В конце должна появиться
строка с количеством установленных пакетов, без слова `error`.

Если в конце есть строки `npm warn` — это не страшно, можно игнорировать.
Если есть `npm error` — пиши мне точный текст ошибки, разберём.

## Шаг 3. Завести базу данных

Приложению нужна база данных PostgreSQL. Для новичка проще всего не ставить её
локально, а взять **бесплатную облачную** — это займёт 3 минуты и не требует установки
ничего лишнего.

### Вариант А (рекомендую): бесплатная база на Neon.tech

1. Зайди на https://neon.tech
2. Зарегистрируйся (можно через GitHub/Google аккаунт)
3. После регистрации тебе предложат создать проект — назови как хочешь, например `crm-masters`
4. На странице проекта найди **Connection string** — это строка вида:
   ```
   postgresql://имя:пароль@xxxxx.neon.tech/neondb?sslmode=require
   ```
5. Скопируй её целиком — она нам понадобится на следующем шаге.

### Вариант Б: поставить PostgreSQL на свой компьютер

Если хочешь именно локально — поставь https://www.postgresql.org/download/, при установке
запомни пароль, который задашь для пользователя `postgres`. Тогда строка подключения будет:
```
postgresql://postgres:ТВОЙ_ПАРОЛЬ@localhost:5432/crm_masters
```
(базу `crm_masters` нужно будет создать через pgAdmin, который ставится вместе с Postgres)

Если не уверен — выбирай Вариант А, он проще для первого раза.

## Шаг 4. Настроить файл переменных окружения

В папке проекта есть файл `.env.example`. Нужно сделать его копию с именем `.env`
(без `.example`).

В терминале:
```
cp .env.example .env
```
(на Windows PowerShell, если `cp` не сработает, используй: `copy .env.example .env`)

Теперь открой файл `.env` в VS Code (просто кликни на него в списке файлов слева)
и отредактируй:

```
DATABASE_URL="сюда вставь строку подключения из Шага 3"
AUTH_SECRET="любой случайный набор символов, см. ниже как сгенерировать"
TELEGRAM_BOT_TOKEN="можно пока оставить пустым или удалить эту строку"
APP_URL="http://localhost:3000"
```

Чтобы сгенерировать `AUTH_SECRET`, введи в терминале:
```
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
```
Скопируй то, что выведется, и вставь как значение `AUTH_SECRET`.

Сохрани файл (Ctrl+S / Cmd+S).

## Шаг 5. Создать таблицы в базе данных

Эта команда читает файл `prisma/schema.prisma` и создаёт по нему все таблицы
в твоей базе данных:

```
npx prisma migrate dev --name init
```

Если попросит название миграции — просто нажми Enter (имя `init` уже передано флагом).

Если всё прошло успешно, увидишь что-то вроде "Your database is now in sync with your schema".

## Шаг 6. Заполнить базу тестовыми данными

Это создаст первого пользователя-администратора и базовые справочники (источники заявок,
виды работ):

```
npm run seed
```

В конце должно появиться:
```
Seed завершён. Логин: admin@crm.local / admin123
```

Запомни эти данные — это логин и пароль, под которыми ты впервые зайдёшь в систему.

## Шаг 7. Запустить приложение

```
npm run dev
```

В терминале появится что-то вроде:
```
▲ Next.js 15.1.0
- Local: http://localhost:3000
```

Это значит, что приложение запущено. **Не закрывай этот терминал** — пока он открыт,
приложение работает. Если закроешь — приложение остановится.

Открой браузер и перейди по адресу:

## Шаг 9. Создать остальных пользователей (Куратор, Менеджер, Мастера)

Сейчас в системе есть только один пользователь — Управляющий (admin). Чтобы добавить
мастеров — зайди в раздел **«Мастера»** в шапке сайта, там есть кнопка «Добавить мастера».

Для Куратора и Менеджера специальной формы пока нет (это создавалось вручную через seed
для admin) — проще всего временно добавить их прямо в базу данных через Prisma Studio,
визуальный редактор базы:

```
npx prisma studio
```

Откроется страница в браузере (обычно http://localhost:5555), где можно открыть таблицу
`User` и добавить новую строку вручную, либо отредактировать существующую.
Поле `role` должно быть одно из: `ADMIN`, `CURATOR`, `MANAGER`, `MASTER`.
Поле `passwordHash` нужно сгенерировать — проще всего временно скопировать значение
у существующего пользователя `admin@crm.local` (тогда пароль будет тот же `admin123`),
а потом через интерфейс приложения это не меняется (смены пароля пока нет в UI — это
зафиксировано в README как нереализованная функция).

## Если что-то не работает

### `npm install` выдаёт ошибки
Скопируй текст ошибки полностью и пришли мне — разберём вместе. Частая причина —
слишком старая версия Node.js (проверь `node -v`, должно быть 18 или выше).

### `npx prisma migrate dev` ругается на подключение к базе
- Проверь, что `DATABASE_URL` в `.env` скопирован полностью, без лишних пробелов
- Если используешь Neon — убедись, что в строке подключения есть `?sslmode=require` в конце
- Если используешь локальный Postgres — проверь, что сервис Postgres запущен

### `npm run dev` запускается, но страница в браузере белая/ошибка
Открой консоль браузера (клавиша F12 → вкладка Console) и посмотри текст ошибки —
пришли мне, разберём.

### Сборка падает с ошибками TypeScript
Это вполне вероятно — весь код писался без проверки компилятором (см. мой предыдущий
ответ). Если увидишь ошибки при `npm run dev` или `npm run build` — пришли мне текст
ошибки целиком, я поправлю код.

### "Module not found" или похожие ошибки про пакеты
Попробуй удалить папку `node_modules` и файл `package-lock.json`, затем заново:
```
npm install
```

## Что дальше, когда заработает

1. Прогони `npm run build` — это найдёт ошибки, которые не видны в режиме разработки
   (`npm run dev` более снисходителен к некоторым проблемам)
2. Попробуй пройти весь сценарий руками: создать заявку → назначить мастера → зайти под
   мастером → принять → на адресе → загрузить фото → завершить → проверить, что в таблице
   заявок появились правильные суммы
3. Когда всё проверено локально — встанет вопрос деплоя (выложить в интернет, чтобы
   мастера заходили с телефонов). Это отдельный следующий шаг, не делай его, пока не
   убедился, что всё работает локально.

Если на любом шаге что-то пойдёт не так — присылай мне точный текст ошибки и на каком
шаге это произошло, я помогу разобраться.
